# Release QA — «Горы и Город — 2026»  
**Дата фиксации:** 2026-05-08

## Готовность

Сайт **готов к локальному просмотру** как статический лендинг: корень проекта — рабочая точка входа, относительные пути к `styles.css`, `main.js` и `assets/` согласованы.

## Проверенные файлы

- `index.html` — структура, заголовки (один `h1`, секции `h2`, пункты программы `h3`), табы (`tablist` / `tab` / `tabpanel`), подписи формы, `aria-hidden` панелей дней, подвал (исправленная иерархия `h2` в блоках с классом `footer-heading`).
- `styles.css` — токены `:root`, glass/радиусы/тени, адаптив, `html { overflow-x: hidden; }` осознанно.
- `main.js` — мобильное меню, Escape, `aria-*`, переключение дней программы, клавиатура табов, валидация контактной формы, без служебных `console.log`.
- `google-form-config.js` — конфигурация для встраивания / отправки (часть параметров ещё пустая).
- `assets/gigi-design-system.svg` — логотип/бренд в шапке.
- `assets/afisha-kit/hero-gelendzhik-arena.png` — слой фона из `--hero-photo` (файл по пути из CSS должен существовать).

## Найденные проблемы и исправления

| Проблема | Исправление |
|----------|-------------|
| Риск отсутствия файла hero по пути из CSS (404 слоя фона) | Файл по пути восстановлен технически; **финальный кадр афиши** нужно подложить **тем же именем** `hero-gelendzhik-arena.png`. |
| В подвале смысловые блоки были `h3` при ожидании `h2` для outline | В `index.html` заголовки подвала заменены на `h2` с сохранением класса `footer-heading` (визуально без изменений). |

## Визуальная ДНК

**Не менялась** в рамках финальной фиксации QA: палитра, стекло, неон, радиусы и язык карточек остаются заданными существующей системой токенов. Подвал — только смена уровня заголовка для доступности/outline. Восстановление hero-фона — техническое (корректная загрузка слоя), не введение нового визуального стиля; итоговое изображение заменяется авторским файлом с тем же именем.

## Background scene update

- Используется **`assets/afisha-kit/hero-gelendzhik-arena.png`** (подключается через **`var(--hero-photo)`** в `:root`).
- Фон — **единая fixed-сцена** на весь лендинг (`position: fixed`, `inset: 0`, `z-index: -2`), а не только для блока hero.
- **`body`** с **`background: transparent`**, чтобы не перекрывать сцену; запасной цвет полотна — **`html { background-color: var(--color-bg); }`**.
- **`.page-bg`** задаёт **два gradient overlay** (90° и 180°) **плюс** слой **`var(--hero-photo)`**; **`background-size: cover`**, базово **`background-position: center center`** для всех слоёв.
- **`.page-bg::after`** удалён как лишний отдельный слой поверх фото.
- На **mobile** добавлено позиционирование только для **фото** (третий слой), чтобы не терять объект арены при `cover`:
  - **`max-width: 768px`:** `background-position: center center, center center, 62% center;`
  - **`max-width: 480px`:** `background-position: center center, center center, 68% center;`
- **Градиенты** остаются **по центру**; **смещается только фото**.
- Весь лендинг **скроллится поверх одной общей** fixed-сцены.

## Не подключено / заглушки перед продом

- **Google Form:** в `google-form-config.js` пустые или тестовые значения для полной `fetch`-отправки; при настроенном `embedUrl` возможен сценарий с iframe. Нужны реальные `formAction` / entry IDs или рабочий embed.
- **Аналитика:** счётчики (например, Метрика / GA) не подключены.
- **Соцсети:** ссылки ведут на базовые домены (`t.me`, `vk.com`, `youtube.com`) без конкретных каналов — заменить на боевые URL.
- **Карта:** используется реальный embed (OpenStreetMap); при необходимости заменить на продакшен-виджет или координаты площадки.

## Локальный запуск

Из каталога проекта:

```bash
cd gory-i-gorod && python3 -m http.server 8080
```

Открыть: `http://localhost:8080`

Альтернатива:

```bash
cd gory-i-gorod && npx --yes serve -l 8080
```
